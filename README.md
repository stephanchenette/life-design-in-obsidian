# Life Design in Obsidian

Sanitized companion materials for a Life Design process implemented in Obsidian.

This repo is intentionally small. It focuses on three things:

- a framework for thinking across the major areas of life
- templates for recurring review and category design
- prompts for turning raw material into structured outputs
- examples that show how the system can be used without exposing private life details

## Structure

- `framework/`
  - conceptual documents
  - interview guide
- `prompts/`
  - report-generation prompt
- `templates/`
  - reusable Markdown and Dataview-based templates
- `examples/`
  - sanitized sample outputs

## Recommended starting point

If you are trying to adapt this to your own vault, start here:

1. Read `framework/life-design-framework-overview.md`
2. Read `framework/lifebook-interview-guide.md`
3. Use `prompts/lifebook-report-generation-prompt.md` after the interview is recorded or transcribed
4. Copy `templates/life-design-category-template.md`
5. Copy `templates/monthly-life-checkup-template.md`
6. Adapt `templates/tracked-metrics-template.md`
7. Review the example files

## Suggested Workflow

1. Create or adapt your 12 category pages.
2. Run the interview using `framework/lifebook-interview-guide.md`.
3. Record or transcribe the interview.
4. Use `prompts/lifebook-report-generation-prompt.md` to turn the transcript into a report.
5. Review the report for missing or thin categories.
6. Convert the strongest strategies into monthly actions.
7. Track ratings and notes over time with the monthly checkup and metrics template.

## Notes

- These materials are framework-level and intentionally sanitized.
- They are not a prescription for how many categories a person must use.
- The examples are illustrative, not autobiographical.
